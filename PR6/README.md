## Опис роботи програми

### Крок 1 — Завантаження ZIP файлів

Програма відкриває ZIP-архіви безпосередньо без розпакування і зчитує `Raw Data.csv` зсередини. Числа у науковому форматі (`1.259E0`) автоматично конвертуються у звичайні float.

```python
with zipfile.ZipFile(filepath, 'r') as z:
    with z.open('Raw Data.csv') as f:
        df = pd.read_csv(f)
```

### Крок 2 — Статистика амплітудних коливань

Для кожного режиму розраховуються:

- **Амплітуда (peak-to-peak)** = max − min — повний розмах коливань
- **Стандартне відхилення** — типова девіація від середнього значення
- **RMS** — ефективне значення прискорення, стандартний показник вібрацій

### Крок 3 — Часові ряди

Графік магнітуди прискорення у часі для кожного режиму. Червона пунктирна лінія — середнє значення. Дозволяє побачити характер, частоту та інтенсивність коливань.

### Крок 4 — Порівняння амплітуд

Стовпчасті діаграми трьох показників: амплітуда peak-to-peak, стандартне відхилення та RMS по кожному режиму. Більше значення відповідає вищій інтенсивності вібрацій.

## Висновок

Експеримент підтвердив, що матрац частково поглинає механічні вібрації — акселерометр фіксував різні рівні прискорення залежно від інтенсивності впливу. Режим спокою показав мінімальну амплітуду, що відповідає фоновому шуму сенсора та гравітаційній складовій. При постукуваннях амплітуда зростала, при інтенсивному натисканні — була максимальною. Показник RMS підтвердив ту саму закономірність. Це демонструє можливість використання акселерометра смартфона як інструменту для вібраційного аналізу та збору реальних фізичних даних без спеціалізованого обладнання.
